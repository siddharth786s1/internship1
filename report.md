# Email Classification and PII Masking Report

## 1. Introduction to the Problem Statement

In the current technological landscape, support teams often receive a large volume of emails that need to be processed efficiently. These emails may contain Personally Identifiable Information (PII), such as names, emails, phone numbers, and payment card details, making it essential to handle them securely while maintaining operational efficiency.

The problem faced by the support team is twofold:
1. **PII Masking:** Identify and mask sensitive information to ensure privacy and compliance with regulations such as GDPR and HIPAA.
2. **Email Classification:** Categorize emails into predefined categories, such as Billing Issues or Technical Support, to enable automated routing and faster resolution.

The goal of this project is to develop an API service that addresses both these challenges by masking PII and classifying emails accurately.

---

## 2. Approach Taken for PII Masking and Classification

### **PII Masking**
The system employs a hybrid approach for PII detection and masking:
1. **SpaCy for Named Entity Recognition (NER):**
   - Utilizes the `en_core_web_sm` SpaCy model to identify entities such as full names (`PERSON`).
   - This provides a robust foundation for detecting contextual PII.

2. **Custom Regex Patterns:**
   - Regular expressions are crafted to detect additional PII types that are not covered by SpaCy, such as:
     - Emails
     - Phone numbers
     - Credit or debit card numbers
     - CVV codes and expiration dates
     - Aadhar numbers and dates of birth (DOB)
   - Detected PII is replaced with placeholders (e.g., `[full_name]`, `[email]`) to anonymize the data.

3. **Masking Pipeline:**
   - The pipeline first uses SpaCy NER to detect entities and then applies regex patterns for comprehensive PII coverage.
   - Overlapping detections are handled to ensure no conflicts in masking.

### **Email Classification**
The classification system uses a machine learning-based pipeline to categorize emails:
1. **Text Preprocessing:**
   - Text is cleaned and normalized to remove unnecessary characters and stopwords.
   - Masked PII placeholders are retained to ensure data privacy during classification.

2. **Feature Extraction:**
   - A **TF-IDF Vectorizer** is employed to convert text data into numerical features suitable for machine learning.

3. **Classification Model:**
   - A **Multinomial Naive Bayes** classifier is used for categorization.
   - This model is lightweight, efficient, and effective for text classification tasks.

4. **Output:**
   - The system outputs the masked email text, a list of detected PII entities, and the predicted category.

---

## 3. Model Selection and Training Details

### **Model Selection**
1. **TF-IDF Vectorizer:**
   - Chosen for feature extraction as it captures the importance of words in the given text while reducing the impact of commonly used words (e.g., "and", "the").
   - Configured with parameters like `max_df=0.95` and `min_df=2` to filter out noise and rare terms.

2. **Multinomial Naive Bayes:**
   - Selected due to its simplicity and efficiency in handling categorical data.
   - Well-suited for text classification tasks with discrete features generated by TF-IDF.

### **Training Process**
1. **Dataset:**
   - The dataset `combined_emails_with_natural_pii.csv` contains email texts and their corresponding categories (e.g., Billing Issues, Technical Support).
   - Text data is preprocessed to clean and normalize the content.

2. **Train-Test Split:**
   - The dataset is split into 80% training and 20% testing sets.
   - Stratified sampling ensures balanced representation of categories in both sets.

3. **Model Training:**
   - The `Pipeline` consists of a TF-IDF Vectorizer followed by a Multinomial Naive Bayes classifier.
   - The model is trained on the cleaned and masked email texts.

4. **Evaluation:**
   - The trained model achieves an accuracy of approximately **69%** on the test set, indicating reasonable performance for the task.

### **Model Saving and Deployment**
- The trained model pipeline is saved as `email_classifier_pipeline.pkl` in the `saved_models/` directory.
- Upon API startup, the model is loaded to enable real-time predictions.

## 4. Conclusion
This project successfully implements a system for PII masking and email classification. By leveraging SpaCy for NER, custom regex for comprehensive PII detection, and a machine learning pipeline for classification, the system addresses the dual challenges faced by support teams. The API exposes this functionality in a user-friendly manner, enabling secure and efficient email processing.
